我们Edge的“集锦”真是太好用啦！

一开始，微软发布Chromium Edge的时候，我心里是拒绝的。老Edge内置的Windows Ink支持在Surface设备上非常好用，而Chromium Edge则不支持。后来，看到微软宣传“集锦”（Collections）功能，我还觉得是不务正业————这东西看起来就只是收集网页链接，同样的事情用收藏夹也可以做。 最近，我开始在台式机上使用Chromium Edge，同时因为服务器过期而折腾了一下Hugo，在这个过程中试着用“集锦”来收集资料。一番使用之后，我对这个功能的印象完全改观了，它真的不只是另一个收藏夹。 首先，添加到“集锦”中的页面，会在标题右侧显示一个缩略图。这样即使是同一个网站上的不同页面，也可以很轻松地分辨。 但是相比于收藏夹，这只是锦上添花，并不算什么大的改进。真正让集锦与收藏夹不同的，我觉得是便签/注释功能（这两个词在“集锦”中某种程度上混用了，下面统称为“便签”）————便签可以附到“集锦”中的页面上，也可以独立存在。独立便签甚至提供了类似Windows便签的编辑器，可以对文本样式进行简单的编辑，或者更改便签的颜色。便签的存在极大方便了对“集锦”中内容的管理，比如贴到页面上来注释收集这个页面的原因、页面关键内容的摘要，或者在独立便签里写一些总结或规划。下面的“集锦”，就是我在创建这个Hugo网站的时候收集的资料。 可以看到，我给关于“使用Github Actions自动部署Hugo到Github Pages”的文章添加了便签，提醒自己文章中的action属性需要更新；给关于“使用Cloudflare来代理Github Pages并且开启源站SSL验证”的文章添加了便签，直接摘录了迁移域名过程中的关键操作；最后，我还添加了一条独立便签，提醒自己在写博客的时候记得使用“Page Bundle”这个功能。 此外，“集锦”还支持多种排序方式，以及分享到Word、Excel和OneNote，这里就不细说了。 总之，“集锦”看起来简单，却很好用，而且潜力也很大。如果微软能为其增加更多的输入和输出方式，比如在手机上支持通过Widget快速添加便签，在分享菜单中添加“分享到集锦”，或是在桌面上直接截取部分网页（Web Clipping），以及支持分享到更多软件、App，我觉得像Flomo这样的笔记软件，可能都没有什么存在的必要了。

用Hugo构建新博客

跟熊壕合租的服务器忘了续费，以前的博客没了。索性来用免费的Github Pages，而静态网站生成器则用Hugo。当然，Cloudflare也要继续用。 使用静态网站生成器+Github Pages有很多种方式，有的是将整个博客的源码和生成后的内容都放在一个分支下，以目录区分，有的是用两个分支（比如master和gh-pages）。但是做了一些功课之后，我选择将源码和生成的内容保存在两个repo，并将“生成”这一步交给Github Actions。 准备Github Repository 创建Repository 在Github上创建两个repo： 一个用来存储Github Pages，名字是username.github.io，例如我的是dantewang.github.io 一个用来存储Hugo网站的源文件，可以是private的，名字随意 配置Actions 参考Github提供的教程，创建SSH Key： ssh-keygen -t ed25519 -C "your_email@example.com" -f path/to/file 这里指定了不同的路径和文件名，以免覆盖git命令用来连接Github的Key。 然后，分别配置两个repo： 在Github Pages repo的 Settings -> Deploy Key 中 Add new secrets，填入公钥文件（*.pub）的内容 在Hugo源文件repo的 Settings -> Secrets 中 Add new secrets，取名为ACTIONS_DEPLOY_KEY，填入私钥文件的内容 git clone第二个repo，添加.github/workflows/gh_pages.yml，内容如下： name: hugo publish on: push: branches: - master jobs: build-deploy: runs-on: ubuntu-18.04 steps: - uses: actions/checkout@v2 with: submodules: true fetch-depth: 0 - name: Setup Hugo uses: peaceiris/actions-hugo@v2 with: hugo-version: 'latest' extended: true - name: Build run: hugo --minify - name: Deploy uses: peaceiris/actions-gh-pages@v3 with: deploy_key: ${{ secrets....
